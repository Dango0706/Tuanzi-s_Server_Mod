# 统计模块 - 产品需求文档

## 概述
- **摘要**：开发一个综合性统计模块作为服务器模组的子项目，用于准确跟踪和管理游戏中的各种指标，同时为玩家和管理员提供必要的功能。
- **目的**：为服务器提供全面的游戏统计数据，帮助管理员了解服务器运行状态，同时为玩家提供个人游戏数据和排行榜功能。
- **目标用户**：Minecraft 服务器管理员和玩家。

## 目标
- 实现全面的游戏统计数据收集和管理功能
- 提供玩家友好的统计数据查询界面
- 支持管理员创建和管理统计数据排行榜
- 确保数据的持久性和一致性
- 优化数据存储和查询性能，减少服务器负载

## 非目标（范围外）
- 不提供实时数据可视化界面
- 不支持自定义统计指标的创建
- 不实现与外部系统的数据同步

## 背景与上下文
- 该模块是基于 Fabric 模组框架开发的服务器模组子项目
- 针对 Minecraft 26.1 版本设计
- 需要与现有的服务器模组架构无缝集成

## 功能需求

### 核心统计需求
- **FR-1**：玩家在线时间跟踪
  - 记录每个玩家的总在线时长（秒、分钟、小时）
  - 实现适当的整数处理以防止溢出问题
  - 存储时间值并提供适当的精度和转换方法

- **FR-2**：服务器运行时间监控
  - 跟踪和记录服务器总运行时间
  - 实现持久存储，在服务器重启后保持准确的运行时间

- **FR-3**：方块交互统计
  - 跟踪每个玩家放置和破坏的方块
  - 在两个级别记录数据：总计数和按方块类型的细分
  - 实现高效的存储结构用于方块类型分类

- **FR-4**：玩家移动跟踪
  - 计算并存储每个玩家旅行的总距离（米）
  - 实现准确的距离计算算法

- **FR-5**：战斗统计
  - 跟踪每个玩家的实体击杀和死亡
  - 在两个级别记录数据：总计数和按实体类型的细分
  - 包括玩家发起的击杀和玩家死亡

- **FR-6**：物品耐久度使用
  - 监控和记录所有工具和盔甲的耐久度消耗
  - 在单个物品级别跟踪使用情况，具有适当的物品识别

### 功能需求
- **FR-7**：玩家统计数据访问
  - 实现玩家可访问的命令以查看个人统计数据
  - 实现查看其他玩家统计数据的功能（具有适当的权限）
  - 设计用户友好的统计数据输出格式

- **FR-8**：管理员计分板管理
  - 开发管理员命令以将特定统计数据显示为计分板
  - 实现计分板创建、修改和删除功能
  - 确保计分板准确反映当前统计数据

- **FR-9**：计分板显示配置
  - 创建管理员功能，在特定游戏位置放置持久计分板显示
  - 实现物理计分板显示的正确渲染和更新机制

- **FR-10**：计分板轮换系统
  - 开发可配置的计分板轮换功能
  - 允许管理员设置轮换间隔并选择要包含在轮换中的统计数据
  - 实现轮换计分板之间的平滑过渡

- **FR-11**：数据持久性和完整性
  - 实现健壮的数据保存机制，在正常和意外服务器关闭时触发
  - 开发定期自动保存功能以防止数据丢失
  - 确保服务器重启和崩溃后的数据一致性

## 非功能需求
- **NFR-1**：性能
  - 统计数据收集和存储不应显著影响服务器性能
  - 数据查询和计分板更新应在 100ms 内完成
  - 内存使用应保持在合理范围内，不超过 50MB

- **NFR-2**：可靠性
  - 系统应能够处理服务器崩溃和意外关闭
  - 数据保存失败时应提供适当的错误处理和日志记录
  - 应实现数据备份机制，防止数据损坏

- **NFR-3**：可扩展性
  - 系统应设计为易于添加新的统计指标
  - 存储结构应支持未来可能的统计数据增长

- **NFR-4**：安全性
  - 确保玩家只能访问有权限查看的统计数据
  - 防止数据操纵和作弊

## 约束
- **技术**：
  - Minecraft 版本：26.1
  - Java 版本：25
  - Fabric Loader：0.18.4
  - Fabric API：0.144.3+26.1

- **依赖**：
  - Fabric API 核心功能
  - 可能需要额外的存储库依赖

## 假设
- 服务器有足够的存储空间来存储统计数据
- 服务器运行环境稳定，能够支持定期数据保存
- 玩家数量在合理范围内（最多 100 名并发玩家）

## 验收标准

### AC-1：玩家在线时间跟踪
- **给定**：玩家连接到服务器
- **当**：玩家在线时
- **然后**：系统应准确记录玩家的在线时间，并在玩家断开连接时保存
- **验证**：`programmatic`
- **说明**：在线时间应精确到秒，并在服务器重启后保持一致

### AC-2：服务器运行时间监控
- **给定**：服务器启动
- **当**：服务器运行时
- **然后**：系统应持续跟踪服务器运行时间，并在服务器重启后累积
- **验证**：`programmatic`

### AC-3：方块交互统计
- **给定**：玩家放置或破坏方块
- **当**：方块放置或破坏事件发生时
- **然后**：系统应记录方块类型和数量，并在两个级别（总计和按类型）更新统计数据
- **验证**：`programmatic`

### AC-4：玩家移动跟踪
- **给定**：玩家在游戏中移动
- **当**：玩家位置发生变化时
- **然后**：系统应计算移动距离并累加到玩家的总距离
- **验证**：`programmatic`
- **说明**：距离计算应使用欧几里得距离公式，单位为米

### AC-5：战斗统计
- **给定**：玩家参与战斗
- **当**：玩家击杀实体或被实体击杀时
- **然后**：系统应记录击杀和死亡事件，并在两个级别（总计和按实体类型）更新统计数据
- **验证**：`programmatic`

### AC-6：物品耐久度使用
- **给定**：玩家使用工具或盔甲
- **当**：物品耐久度减少时
- **然后**：系统应记录耐久度消耗，并按物品类型分类
- **验证**：`programmatic`

### AC-7：玩家统计数据访问
- **给定**：玩家执行统计命令
- **当**：玩家输入 `/stats` 或 `/stats <player>` 命令时
- **然后**：系统应显示相应的统计数据，格式清晰易读
- **验证**：`human-judgment`

### AC-8：管理员计分板管理
- **给定**：管理员执行计分板命令
- **当**：管理员输入计分板管理命令时
- **然后**：系统应创建、修改或删除计分板，并显示正确的统计数据
- **验证**：`human-judgment`

### AC-9：计分板显示配置
- **给定**：管理员放置计分板显示
- **当**：管理员在特定位置放置计分板显示时
- **然后**：系统应在该位置创建物理计分板显示，并定期更新数据
- **验证**：`human-judgment`

### AC-10：计分板轮换系统
- **给定**：管理员配置计分板轮换
- **当**：轮换间隔到达时
- **然后**：系统应平滑切换到下一个计分板，显示不同的统计数据
- **验证**：`human-judgment`

### AC-11：数据持久性和完整性
- **给定**：服务器正常或意外关闭
- **当**：服务器关闭或崩溃时
- **然后**：系统应保存所有统计数据，并在服务器重启后恢复
- **验证**：`programmatic`
