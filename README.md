# Tuanzi's Server Mod (团子服务器增强模组)

这是一个为 Fabric 26.1 打造的全能型服务器管理模组。它深度整合了正盗版混合验证、多币种动态经济体系、可视化智能商店以及全方位的玩家行为统计，旨在为中大型服务器提供专业且便捷的管理方案。

---

## 🛡️ 身份验证模块 (Auth) - 深度详解

本模块提供了一套严密的身份校验机制，完美兼容正版与离线（盗版）玩家。

### 1. 自动账户识别逻辑
*   **正版玩家 (Online)**：
    *   **判定依据**：UUID 版本为 4 且通过 Mojang 官方验证。
    *   **特性**：**静默登录**。系统自动识别并授予所有权限，玩家无需任何额外操作。
*   **离线玩家 (Offline)**：
    *   **判定依据**：UUID 版本为 3。
    *   **安全链条**：进入服务器后会进入“受限状态”（无法移动、无法发言、无法交互、无法受伤），需在限定时间内完成登录。
    *   **离线 UUID 固定**：模组内置离线 UUID 生成算法，确保即便玩家修改名字（在离线模式下），其白名单和数据资产依然能通过原始生成的 UUID 找回。

### 2. 高级安全特性
*   **会话 (Session) 与 IP 持久化**：玩家在成功登录后，系统会记录其 IP 地址。在配置文件指定的有效期内（如 24 小时），只要玩家 IP 未变，再次进服将触发 **自动登录**。
*   **暴力破解防护**：内置登录尝试限制，多次输入错误密码将触发临时封禁。

## 🏪 智能箱子商店 (Shop) - 交互指南

箱子商店是本模组的核心特色，它将传统的命令式商店转化为直观的视觉化交互。

### 1. 引导式创建全流程
1.  **物理准备**：放置一个箱子，并在其正面贴上告示牌。
2.  **触发识别**：手持物品点击告示牌。
3.  **智能交互**：
    *   **步骤 A (类型)**：聊天框弹出 `[出售]` 和 `[收购]` 按钮，点击即可设定商店性质。
    *   **步骤 B (单价)**：在聊天框输入数字（如 `50.5`）。
    *   **步骤 C (备注)**：输入该商店的宣传语或说明（输入 `{None}` 可留空）。
4.  **自动渲染**：创建完成后，商店上方会生成一个**悬浮的掉落物模型**，告示牌会自动涂蜡。

### 2. 动态定价 (Dynamic Pricing) 算法模型
基于市场压力的动态定价模型公式如下：
*   **收购店 (BUY)**：$$P = x + (y - x) \times \frac{K}{K + S}$$
*   **出售店 (SELL)**：$$P = x + (y - x) \times \frac{S}{K + S}$$
*   **变量解析**：
    *   $x$：最低价 | $y$：最高价 | $K$：半衰常数 (调节价格敏感度) | $S$：系统实时库存/热度值。
*   **价格平复**：系统每 24000 tick 会根据商店独立的 `decayRate`（默认 1%）自动扣除 $S$ 值，使价格随时间自然恢复到初始水平。

### 3. 核心物理防护
*   **防爆机制**：所有商店方块（箱子/告示牌）对 TNT 及 Creeper 等爆炸**完全免疫**。
*   **展示物锁定**：悬浮物品由 `ItemEntity` 驱动，但应用了 `noPhysics` 和 `noGravity` 属性，**无法被水流冲走、无法被漏斗吸取、玩家无法拾取**。
*   **自动涂蜡保护**：所有由商店系统生成的告示牌会自动进行**化学涂蜡**，任何非管理员玩家均无法通过交互修改告示牌内容。

## 💰 经济系统 API (Economy API)

模组为其他开发者提供了一套稳健的经济交互接口，支持多货币管理。

### 1. 获取 API 实例
可以通过 `EconomyAPIImpl.getInstance()` 或模块管理器获取 API 接口。

### 2. 核心方法说明
| 方法名 | 说明 |
| :--- | :--- |
| `getBalance(UUID, String)` | 获取玩家在特定钱包（货币 ID）中的余额 |
| `deposit(UUID, String, double)` | 向玩家钱包存入指定金额 |
| `withdraw(UUID, String, double)` | 从玩家钱包扣除指定金额 |
| `hasEnough(UUID, String, double)` | 检查玩家是否有足够的余额执行某项操作 |
| `transfer(UUID, UUID, String, double)` | 执行玩家间的转账（原子操作，确保资金安全） |
| `registerWalletType(String, Component)` | 动态注册一种新的货币类型（如：金币、硬币、贡献值） |
